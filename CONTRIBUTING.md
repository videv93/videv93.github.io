# Contributing Guidelines

Thank you for your interest in contributing to syscraft906 projects!

## Getting Started

1. Fork the repository
2. Create a feature branch: `git checkout -b feature/your-feature`
3. Make your changes
4. Write/update tests
5. Commit with clear messages: `git commit -m "feat: add your feature"`
6. Push to your fork
7. Create a Pull Request

## Standards

### Code Quality
- Follow existing code style
- Write clear, descriptive variable names
- Add comments for complex logic
- Aim for >80% test coverage

### Commit Messages
```
<type>: <subject>

<body>

<footer>
```

**Types**: feat, fix, docs, style, refactor, test, chore

**Example**:
```
feat: add OCO order support

Implements one-cancels-other order functionality for KitKat Exchange.
Includes validation and risk checks.

Closes #123
```

### Pull Requests
- Reference related issues
- Add screenshots/demos for UI changes
- Update documentation
- Wait for 1 approval before merging

## Branch Protection

Main branches are protected:
- ✅ 1 approval required
- ✅ Status checks must pass
- ✅ No direct pushes

## Community

- Respect diversity and inclusion
- Be constructive in feedback
- Ask questions in Discussions
- Report issues responsibly

## Questions?

- 📖 Check [README.md](./README.md)
- 💬 Use Discussions tab
- 🐛 Open an issue

Happy contributing! 🎉
